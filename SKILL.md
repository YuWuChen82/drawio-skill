# drawio-skill

根据自然语言生成标准 `.drawio` XML 文件（兼容 draw.io / diagrams.net）。布局结构化、网格对齐（10px）。

## 何时激活

以下场景自动激活本 skill：

- 用户说"画图"、"帮我画"、"生成图"
- 提及 "draw.io"、"diagrams.net" 或要求导入 draw.io
- 请求"ER图"、"用例图"、"时序图"、"流程图"、"类图"
- 描述业务过程、数据库结构、系统交互或用户需求
- 提供表结构、字段、工作流或角色-用例描述

## 支持的图类型

| 类型 | 布局方向 | 核心元素 |
|------|---------|---------|
| **ER 图** | 网格（左→右 或 上→下） | 表 + 字段行 + 关系连线 |
| **用例图** | 左（角色）→ 中（系统边界）→ 右 | 角色图标 + 系统框 + 用例 + 连线 |
| **时序图** | 垂直，上→下 | 对象框 + 生命线 + 消息箭头 |
| **流程图/BPMN** | 左→右（或上→下） | 圆角矩形（开始/结束）+ 矩形（活动）+ 菱形（判断）+ 箭头 |
| **类图** | 网格布局 | 类框（名/属性/方法）+ 关系连线 |

## 执行步骤

### 步骤 1 — 确认类型和范围

若用户需求不明确，主动询问：

```
1. 图的类型：[ER 图 / 用例图 / 时序图 / 流程图 / 类图 / 其他]
2. 主要对象：[角色 / 系统 / 模块 / 数据表]
3. 核心内容：
   - ER：表列表 + 字段 + 关系
   - 用例图：角色列表 + 用例列表 + include/extend/泛化
   - 时序图：参与者 + 各参与者的操作 + 消息/返回
   - 流程图：开始 → 步骤 → 判断 → 结束
4. 输出文件名：[默认：{场景}_{图类型}.drawio]
```

### 步骤 2 — 布局原则：语义分层 + 分组（核心）

> **布局不仅是公式，更是语义。元素的位置由其含义决定，而非序号。**

#### 通用视觉层级

```
第一优先级（主视觉区）：核心业务流程、主实体、主参与者
    ↓
第二优先级（次要区）：管理/配置功能、次要参与者
    ↓
第三优先级（边缘区）：异常处理、边缘用例、辅助功能
```

- 核心元素居中或居左偏上；边缘元素靠边
- 同级元素均匀分布，不同级元素用留白区分
- **禁止**：所有元素平铺在一条水平线上；包含/扩展关系两端相距甚远

---

#### ER 图 — L1~L4 语义分层布局

**UML参考：** 实体按依赖层次从左上→右下排列，主键关系路径清晰。

```
布局结构：

  L1 基础实体 ───────────────────────────▶ L2 业务实体
  （用户/账户/角色）                         （商品/订单/帖子）
  左上角 x=40                                居中

  L3 附属/明细                              L4 配置/字典
  （订单项/评价/收货地址）                    （分类/标签/优惠券/权限）
  右部或下行                                 右下边缘

  图例 ──────────────────────────────────────────▶

公式：
  行间距 = max(所有表高) + 80
  列宽   = max(所有表宽) + 80
  L层确定行号：L1→行0，L2→行1，L3→行2，L4→行3
  表x = 40 + 列号 × 列宽
  表y = 40 + L层行号 × 行间距
```

- **禁止**：L1表放右下；L4表放左上；跨层跳跃连线（L4直连L1）；表间距 < 60px
- 图例：固定在右下角 x=1140, y=40

#### 用例图 — 角色-用例分组 + 主次分层布局

**UML参考（UML 2.5规范）：** Actor 在边界外侧，用例在边界内侧；Include/Extend 关系两端必须相邻；用例按主次在垂直方向分层。

**三层分区（从上到下）：**

```
┌──────────────────────────────────────────────────────────────┐
│ 第1层（主区）：主Actor + 核心业务用例（水平排列，紧凑分组）    │
│  act1 ─────── uc1 ─ uc2 ─ uc3                               │
│  Admin ────── 管理商品 ─ 管理用户 ─ 查看报表                  │
│                                                              │
│  act2 ─────── uc4 ─ uc5                                     │
│  User ─────── 浏览商品 ─ 下单 ───┐                          │
│                                include ─── uc6(验证库存)      │
│                                extend ─── uc7(追加评价)       │
│                                                              │
│ 第2层（次区）：次级Actor + 管理/查询用例（边缘）              │
│  act3 ─────── uc8 ─ uc9                                     │
│  Guest ────── 注册 ──── 登录                                 │
│                                                              │
│ 第3层（外部系统，右边或底部）：外部Actor + 辅助用例            │
│                            act_sys ── uc10(发送邮件)         │
└──────────────────────────────────────────────────────────────┘
  系统边界框：包裹所有用例，按实际范围计算宽高（不得写死900×600）
```

**角色纵向排序（由主到次）：**
```
主Actor（User/Admin）     → x=40，y按行号（100, 180, 260...）
次级Actor（Guest/Vendor） → 主Actor下方
外部系统Actor             → 放在系统边界右侧外面，y对应其用例组
```

**用例分组（组内紧密，组间留白）：**
```
组内间距：≤30px（Include/Extend 两端必须 ≤ 30px）
组间间距：≥80px
用例宽度 = max(文本长度px + 60, 120)（不得写死160px）
用例高度 = 50
```

**Include/Extend 布局规则（关键）：**
- Include UC 放在基UC水平右侧（x_基用例 + 基用例宽 + 20），两者 y 对齐
- Extend UC 放在基UC垂直下方（y_基用例 + 80），x 与基UC对齐
- Include 箭头标注 `<<include>>`，Extend 箭头标注 `<<extend>>`

> **禁止**：把所有用例平铺在一个水平带上；Include/Extend 用例远离基用例（> 30px）；不同 Actor 的用例混杂无分组

#### 时序图 — 对象距离分层布局（BCE 参考模式）

**UML参考：** 从左到右代表"与发起者距离"递减——Actor → 边界/接口 → 控制/服务 → 实体/仓库 → 外部系统。

```
布局结构（从左到右）：

  Actor/User ──▶ 前端UI/接口 ──▶ 业务服务/控制器 ──▶ 数据库/Repo ──▶ 外部系统
  (发起者)       (Boundary)       (Control/Service)    (Entity)           (External)

对象间距：列宽 = max(该列文本最长px + 40, 100)
对象框 y=40，高50
消息y：y = 90 + 消息序号×60（序号1→90，序号2→150...）
生命线：从对象框底部(y=90)延伸至最末消息y+40
```

- 自调用：嵌套在对象生命线上（灰框包裹，y随消息序号递增）

> **禁止**：把数据库和前端随意排列（应有BCE顺序）；外部系统和内部服务混杂

---

#### 流程图 — 泳道分组 + 主干居中布局

**UML参考：** 同一泳道 = 同一角色/同一系统/同一职责；主流程居中，异常路径靠边。

```
垂直泳道布局（按职责）：

  ┌──────────────┬──────────────┬──────────────┐
  │  前端/发起者   │  后端/业务     │  数据/外部    │
  │  ──────────  │  ──────────  │  ──────────  │
  │  输入节点      │  处理节点      │  数据库节点    │
  │  开始椭圆      │  活动矩形      │  结束椭圆      │
  └──────────────┴──────────────┴──────────────┘
  泳道分隔线：strokeWidth=2，垂直分隔

主干流（水平左→右）：
  节点间距：水平≥60px，垂直≥80px
  判断分支：Yes→右(x+宽+60) 或 下(y+高+80)，No→反向
  合并：实心圆，直径30px

异常/错误分支：
  放在主干下方，红色虚线框标注
```

- **禁止**：没有泳道所有活动混在一线；判断节点后缺少Yes/No两个分支；错误路径无标注

---

#### 类图 — 包分组 + 继承树垂直布局

**UML参考：** 同包类在一起；继承树父上子下（垂直）；依赖箭头从客户端指向供应者。

```
布局结构：

  包A（domain层）：
    ┌─────────────────────┐
    │   User (父类)         │  ← 父类在顶部
    │   ─────────────────  │
    │   - id: Long         │
    │   + getId(): Long    │
    └─────────────────────┘
              │
    ┌──────────┴──────────┐
    │ Admin               │ │ Customer          ← 子类在父下方，垂直排列
    └─────────────────────┘ └─────────────────────┘
    （子类与父类列对齐，继承连线垂直向上）

  包B（infrastructure层）：
    ┌──────────────┐  ┌──────────────┐
    │ UserRepo     │  │ OrderRepo    │  ← 同包内横向排列，间距40px
    └──────────────┘  └──────────────┘

  包边界（可选虚线框）：
    strokeColor=#999，strokeDashPattern="3 3"
    宽 = max(包内所有类宽) + 60
    高 = 所有行高 + 80
```

- **禁止**：不同包的类混在一起；继承树横向排列；类间距随意

### 步骤 3 — 格式规范约束（必须遵守）

**宽高公式化原则（核心，禁止硬编码）：**
- 宽度 = max(内容文本长度 + padding, 最小宽度)
- 高度 = minHeight（可固定）或按行数/字段数计算
- 位置 = 起始坐标 + 序号 × (元素尺寸 + 间距)
- 具体公式见"步骤2"各图类型的布局结构说明

> **警告**：任何形状的 `width`、`height` 均不得写死为固定像素值，必须用公式计算。

**画布尺寸规范：**
| 规模 | pageWidth × pageHeight | 适用场景 |
|------|----------------------|---------|
| 小（< 5 个元素） | 800 × 600 | 单角色 / 简单流程 |
| 中（5~15 个元素） | 1200 × 800 | 普通 ER / 用例 / 时序 |
| 大（> 15 个元素） | 1600 × 1200 | 复杂 ER / 多角色时序 |

**连线交叉禁止规则：**
- 优先采用正交（orthogonal）路由
- 若两条连线即将交叉，插入拐点：
  ```xml
  <mxGeometry relative="1" as="geometry">
    <Array as="points">
      <mxPoint x="300" y="200"/>
    </Array>
  </mxGeometry>
  ```
- ER图：`edgeStyle=entityRelationEdgeStyle` + `orthogonalLoop=1`
- 用例图：角色→用例连线必须是直线，不得穿越其他用例

**内容分布规范（禁止挤到一角）：**
- 核心元素居中或左上，边缘元素靠边
- 同行元素均匀分布；某行元素偏少（< 3个）时居中或分散排列
- 各级之间用留白区分

### 步骤 4 — 生成 XML

根据图类型，参照 `references/drawio_xml_format.md` 中的格式规范生成 XML。

### 步骤 5 — 输出

1. 写入文件：`C:\Users\{username}\{filename}.drawio`
2. 回复内容：
   - 图类型和元素概要
   - 核心结构说明
   - 导入方式："draw.io → 文件 → 打开 → 选择 .drawio 文件"
   - 是否有需要调整的地方

## 输出标准

### 通用规则

- 文件路径：`C:\Users\{username}\{filename}.drawio`
- 外层包装：`<mxfile host="app.diagrams.net">`
- 画布：`gridSize="10"`，`guides="1"`，`connect="1"`，`arrows="1"`
- 字体：`fontFamily=Helvetica`
- 所有坐标和尺寸均为 10 的倍数

### ER 图

- 表宽：`max(最长字段行字符数×8 + 20, 180)`（不得写死）
- 表高：`30 + 字段数 × 18`
- 关系连线：`edgeStyle=entityRelationEdgeStyle`
  - 1:N：实线 + `endArrow=ERmany`
  - 1:1：实线 + `endArrow=ERone`
  - 自引用：虚线 + `dashed=1`
- 图例：固定在 x=1140, y=40

### 用例图

- 系统边界：宽高按实际用例范围计算（不得写死900×600）
- 角色：`shape=umlActor`
- 用例椭圆：宽 `max(文本长度+60, 120)`，高50（不得写死）
- 关联：实线，无箭头（`endArrow=none`）
- Include/Extend：虚线，`endArrow=open`，标签 `<<include>>` / `<<extend>>`；Include UC与基UC间距≤30px
- 泛化：实线，`endArrow=classic`（实心三角）
- 角色↔用例间距：≥60px

### 时序图

- 列宽：`max(对象文本最长px + 40, 100)`（BCE顺序，不得写死200px）
- 对象框：y=40，宽按文本计算，高50
- 生命线终点：`y=90 + 消息数×60 + 40`（动态）
- 同步消息：实线 + 实心三角 `endArrow=block`，`endSize=10`
- 异步消息：实线 + 空心三角 `endArrow=open`
- 返回：虚线 + 空心三角

### 流程图

- 活动节点：宽 `max(文本长度+40, 120)`，高按行数计算（不得写死160×60）
- 判断菱形：最小120×80
- 开始/结束椭圆：宽按文本，高固定40
- 连接箭头：`endArrow=classic`
- 分支标签："是/Y" → 右；"否/N" → 下
- 合并：实心圆

### 图例（必选）

每个图在右下角必须包含图例：

```
实线箭头 = 直接关联
虚线箭头 = 依赖/扩展
三角箭头 = 泛化/继承
乌鸦脚   = 一对多

Solid arrow = direct association
Dashed arrow = dependency/extend
Triangle = generalization/inheritance
Crow's foot = one-to-many
```

## 禁止事项

- 不生成用户未要求的图类型
- 不跳过确认步骤
- 不使用非标准 XML 格式
- 不输出纯文本代替 XML 文件
- 不将大量内容塞入单个图（保持可读性）
- **禁止画布滥用**：尺寸远超内容所需；内容堆在画布一角；连线之间相互交叉
- **禁止硬编码宽高**：必须使用公式计算
- **禁止混乱布局**：元素平铺一线；Include/Extend 关系两端分离超过30px；无泳道无分层
