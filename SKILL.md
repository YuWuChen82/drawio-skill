# drawio-skill

根据自然语言生成标准 `.drawio` XML 文件（兼容 draw.io / diagrams.net）。布局结构化、网格对齐（10px）。

## 何时激活

以下场景自动激活本 skill：

- 用户说"画图"、"帮我画"、"生成图"
- 提及 "draw.io"、"diagrams.net" 或要求导入 draw.io
- 请求"ER图"、"用例图"、"时序图"、"流程图"、"类图"
- 描述业务过程、数据库结构、系统交互或用户需求
- 提供表结构、字段、工作流或角色-用例描述

## 支持的图类型

| 类型 | 布局方向 | 核心元素 |
|------|---------|---------|
| **ER 图** | 网格（左→右 或 上→下） | 表 + 字段行 + 关系连线 |
| **用例图** | 左（角色）→ 中（系统边界）→ 右 | 角色图标 + 系统框 + 用例 + 连线 |
| **时序图** | 垂直，上→下 | 对象框 + 生命线 + 消息箭头 |
| **流程图/BPMN** | 左→右（或上→下） | 圆角矩形（开始/结束）+ 矩形（活动）+ 菱形（判断）+ 箭头 |
| **类图** | 网格布局 | 类框（名/属性/方法）+ 关系连线 |

## 执行步骤

### 步骤 1 — 确认类型和范围

若用户需求不明确，主动询问：

```
1. 图的类型：[ER 图 / 用例图 / 时序图 / 流程图 / 类图 / 其他]
2. 主要对象：[角色 / 系统 / 模块 / 数据表]
3. 核心内容：
   - ER：表列表 + 字段 + 关系
   - 用例图：角色列表 + 用例列表 + include/extend/泛化
   - 时序图：参与者 + 各参与者的操作 + 消息/返回
   - 流程图：开始 → 步骤 → 判断 → 结束
4. 输出文件名：[默认：{场景}_{图类型}.drawio]
```

### 步骤 2 — 规划布局（按类型）

**ER 图：**
```
布局公式：
  首行Y = 40
  行间距 = max(所有表高) + 80
  列间距 = max(所有表宽) + 80
  表x = 起始x + 列号 × 列间距
  表y = 起始y + 行号 × 行间距

示例（2行3列，等宽表200px，间距80px）：
  表A → x=40,   y=40
  表B → x=320,  y=40
  表C → x=600,  y=40
  表D → x=40,   y=190  （第二行，高度=表A高+80）
```
- L1 基础实体（User/Account）→ 左上
- L2 业务实体（Product/Order）→ 居中
- L3 附属/明细（OrderItem/Payment）→ 右或下
- L4 配置类（Category/Coupon/Tags）→ 右下边缘
- **禁止**：写死表坐标（如 `TableA(40,40)`）——必须用公式按实际数量计算；连线交叉；表间距 < 60px
- 图例：固定在右下角 x=1140, y=40

**用例图：**
```
布局公式：
  角色x = 40（固定列宽）
  角色y = 100 + 序号×80

  系统边界框:
    x = 角色x + 角色宽(50) + gap(20) = 110
    width = 用例列数 × 列宽(160) + 右边距(40)
    height = 用例行数 × 行高(80) + 上下padding(40)

  用例x = 110 + 列号 × 160
  用例y = 40 + 行号 × 80

用例宽度 = max(文本长度px + 60, 最小120px)（不得写死160px）
用例高度 = 50（椭圆固定高）
```
- `<<include>>`：虚线箭头 + `<<include>>` 标签
- `<<extend>>`：虚线箭头 + `<<extend>>` 标签
- 泛化：实线 + 空心三角
- 关联：实线，无箭头
- **禁止**：用例超出系统边界框；连线穿过用例椭圆；角色与用例间距 < 60px；写死系统框宽高（必须按用例数量计算）

**时序图：**
```
布局公式：
  列宽 = max(对象文本最长px + 40, 100px)（不等宽，不得写死200px）
  列起始x = 40, 40+列宽1, 40+列宽1+列宽2, ...

  对象框:
    y = 40
    width = max(文本长度px + 40, 100)
    height = 50

  消息y坐标: y = 90 + 消息序号×60
    序号1 → y=90, 序号2 → y=150, 序号3 → y=210, ...

  生命线: 从对象框底部(y=90)到最末消息y + 40
```
- 同步请求：实线 + 实心三角 `endArrow=block`
- 异步请求：实线 + 空心三角 `endArrow=open`
- 返回：虚线 + 空心三角
- 自调用：嵌套灰色矩形 + 步骤编号标签
- **禁止**：写死生命线终点（如 `y=600`）——必须按消息数量动态计算；写死列宽为200px

**流程图：**
```
布局公式：
  节点宽度 = max(文本长度px + 40, 最小宽度)
  节点高度 = max(文本行数×20 + 20, 最小高度)
    活动节点最小: 120×40
    判断菱形最小: 120×80

  水平间距 ≥ 60px，垂直间距 ≥ 80px
  判断分支: Yes → 右(x+节点宽+60) 或 下(y+节点高+80)，No → 反向

  节点x = 起始x + (节点序号-1) × (节点宽度 + 60)
  节点y = 起始y + 当前行偏移
```
- **禁止**：写死节点宽高（如 `width="160" height="60"`）；节点重叠；画布留白过多或内容挤在一角

**类图：**
- 三段式框：类名（粗体）/ 属性 / 方法
- 行分隔线 `rowLines=1`
- 类头部蓝色填充

### 步骤 3 — 格式规范约束（必须遵守）

**宽高公式化原则（核心，禁止硬编码）：**
- 宽度 = max(内容文本长度 + padding, 最小宽度)
- 高度 = minHeight（可固定）或按行数/字段数计算
- 位置 = 起始坐标 + 序号 × (元素尺寸 + 间距)
- 具体公式见"步骤 2"各图类型的"布局公式"部分
- 参考 `references/drawio_xml_format.md` 中的"附：常用形状尺寸速查表"

> **警告**：任何形状的 `width`、`height` 均不得写死为固定像素值（如 `width="160" height="50"`），必须用公式计算。

**画布尺寸规范（禁止过大或过小）：**
| 规模 | pageWidth × pageHeight | 适用场景 |
|------|----------------------|---------|
| 小（< 5 个元素） | 800 × 600 | 单角色 / 简单流程 |
| 中（5~15 个元素） | 1200 × 800 | 普通 ER / 用例 / 时序 |
| 大（> 15 个元素） | 1600 × 1200 | 复杂 ER / 多角色时序 |

> **警告**：不要因为内容多就把 pageWidth 设得巨大。应在 1600×1200 内通过合理分布元素来容纳内容，而非扩大画布。
| 规模 | pageWidth × pageHeight | 适用场景 |
|------|----------------------|---------|
| 小（< 5 个元素） | 800 × 600 | 单角色 / 简单流程 |
| 中（5~15 个元素） | 1200 × 800 | 普通 ER / 用例 / 时序 |
| 大（> 15 个元素） | 1600 × 1200 | 复杂 ER / 多角色时序 |

> **警告**：不要因为内容多就把 pageWidth 设得巨大。应在 1600×1200 内通过合理分布元素来容纳内容，而非扩大画布。

**连线交叉禁止规则（所有图类型）：**
- 绘制连线时，优先采用正交（orthogonal）路由，避免斜线
- 若两条连线即将交叉，重新调整端点坐标或插入拐点：
  ```xml
  <mxGeometry relative="1" as="geometry">
    <Array as="points">
      <mxPoint x="300" y="200"/>  <!-- 拐点坐标 -->
    </Array>
  </mxGeometry>
  ```
- ER 图：使用 `edgeStyle=entityRelationEdgeStyle` + `orthogonalLoop=1` 自动避让
- 用例图：角色→用例连线必须是直线，不得穿越其他用例

**内容分布规范（禁止挤到一角）：**
- 元素应沿画布均匀分布，参考"步骤 2"中的行列坐标表
- 若某行元素偏少（< 3个），将元素在该行内居中或分散排列，而非堆在左侧
- 系统边界框内的用例须在框内垂直居中，不得紧贴上边缘或下边缘

### 步骤 4 — 生成 XML

根据图类型，参照 `references/drawio_xml_format.md` 中的格式规范生成 XML。

### 步骤 5 — 输出

1. 写入文件：`C:\Users\{username}\{filename}.drawio`
2. 回复内容：
   - 图类型和元素概要
   - 核心结构说明
   - 导入方式："draw.io → 文件 → 打开 → 选择 .drawio 文件"
   - 是否有需要调整的地方

## 输出标准

### 通用规则

- 文件路径：`C:\Users\{username}\{filename}.drawio`
- 外层包装：`<mxfile host="app.diagrams.net">`
- 画布：`gridSize="10"`，`guides="1"`，`connect="1"`，`arrows="1"`
- 字体：`fontFamily=Helvetica`
- 所有坐标和尺寸均为 10 的倍数（网格对齐）

### ER 图

- 表宽：`max(最长字段行字符数×8 + 20, 180)`（不得写死180/200/220）
- 表高：`30 + 字段数 × 18`
- 关系连线：`edgeStyle=entityRelationEdgeStyle`
  - 1:N：实线 + `endArrow=ERmany`
  - 1:1：实线 + `endArrow=ERone`
  - 自引用：虚线 + `dashed=1`
- 图例：固定在 x=1140, y=40

### 用例图

- 系统边界：宽高按用例数量计算（见步骤2公式），不得写死900×600
- 角色：`shape=umlActor`
- 用例椭圆：宽 `max(文本长度+60, 120)`，高固定50，不得写死160×50
- 关联：实线，无箭头（`endArrow=none`）
- Include/Extend：虚线，`endArrow=open`，标签 `<<include>>` / `<<extend>>`
- 泛化：实线，`endArrow=classic`（实心三角）
- 角色↔用例间距：≥60px

### 时序图

- 列宽：`max(对象文本最长px + 40, 100)`（不等宽，不得写死200px）
- 对象框：y=40，宽按文本长度计算，高50
- 生命线终点：`y=90 + 消息数×60 + 40`（动态，不得写死y=600）
- 同步消息：实线 + 实心三角 `endArrow=block`，`endSize=10`
- 异步消息：实线 + 空心三角 `endArrow=open`
- 返回：虚线 + 空心三角

### 流程图

- 活动节点：宽 `max(文本长度+40, 120)`，高按行数计算（不得写死160×60）
- 判断菱形：最小120×80，可随文本增大
- 开始/结束椭圆：宽按文本，高固定40（不得写死80×50）
- 连接箭头：`endArrow=classic`
- 分支标签："是/Y" → 右；"否/N" → 下
- 合并：实心圆

### 图例（必选）

每个图在右下角必须包含图例：

```
实线箭头 = 直接关联
虚线箭头 = 依赖/扩展
三角箭头 = 泛化/继承
乌鸦脚   = 一对多

Solid arrow = direct association
Dashed arrow = dependency/extend
Triangle = generalization/inheritance
Crow's foot = one-to-many
```

## 禁止事项

- 不生成用户未要求的图类型
- 不跳过确认步骤
- 不使用非标准 XML 格式
- 不输出纯文本代替 XML 文件
- 不将大量内容塞入单个图（保持可读性）
- **禁止**：画布尺寸远超内容所需；内容堆在画布一角；连线之间相互交叉
- **禁止写死宽高**：不得用固定像素值（如 `width="160"`）作为任何形状的宽高；必须使用公式计算（见步骤2"布局公式"和步骤3"宽高公式化原则"）
- 不产生混乱布局（禁止重叠、最小化交叉）
